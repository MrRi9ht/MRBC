# 项目 PRD：单机德州扑克小游戏

## 项目概述

本项目是一个基于 Web 端的单机德州扑克（Texas Hold'em）休闲游戏。核心体验是模拟真实的线下扑克牌局，用户将与 3-5 个具有不同性格设定的 AI 机器人进行对战。游戏主打流畅的交互手感、清晰的牌局逻辑以及轻松解压的单机娱乐氛围。

**技术栈要求：**

- **前端框架：** Vue 3 或 React（推荐使用 Vite 构建）
- **样式方案：** Tailwind CSS（快速构建牌桌与卡牌 UI）
- **动画库：** GSAP 或 Framer Motion（用于发牌、筹码移动、底池汇聚等动画）
- **状态管理：** Pinia 或 Zustand（管理复杂的牌局状态、玩家资金与下注轮次）

## 核心里程碑拆解

### Phase 1: 牌桌环境搭建与基础 UI 绘制

**目标：** 还原经典的绿色绒布牌桌视觉，完成玩家座位布局与基础交互按钮。

**具体任务：**

1. 初始化前端项目并配置 Tailwind CSS。
2. 绘制核心牌桌区域：包括椭圆形牌桌背景、中央底池（Pot）显示区、公共牌（Community Cards）摆放区。
3. 布局玩家座位：设定 6 人桌布局，包含"自己"（底部居中）与 5 个 AI 机器人（分布在牌桌四周）。
4. 实现基础 UI 组件：玩家头像框、筹码数量显示、当前下注额气泡、庄家按钮（Dealer Button）标识。
5. 开发玩家操作面板：包含"弃牌（Fold）"、"过牌/跟注（Check/Call）"、"加注（Raise）"按钮，并根据游戏状态控制按钮的禁用与高亮。

### Phase 2: 核心扑克逻辑引擎与发牌动画

**目标：** 建立完整的德州扑克规则引擎，实现从洗牌到翻牌圈的流程，并配合流畅的动画。

**具体任务：**

1. 编写扑克核心逻辑类：实现 52 张牌的洗牌算法、发牌顺序控制、以及牌型大小比对算法（如皇家同花顺 > 同花顺 > 四条等）。
2. 定义牌局状态机：严格管理游戏阶段流转，包括"等待开始 -> 前注 -> 翻牌前 -> 翻牌 -> 转牌 -> 河牌 -> 摊牌比大小 -> 结算"。
3. 实现发牌动画：使用动画库实现卡牌从牌堆平滑飞向各个玩家手牌区（Hole Cards）以及公共牌区的视觉效果，需带有翻转卡牌的动画细节。
4. 完成前两轮交互：实现玩家拿到底牌后的决策逻辑，以及 AI 机器人根据简单随机策略进行跟注或弃牌的流程。

### Phase 3: 完整下注轮次与 AI 行为系统（核心升级）

**目标：** 跑通整局游戏流程，**赋予 AI 拟人化的性格与博弈能力**，实现底池与筹码的动态变化。

**具体任务：**

1. 完善下注逻辑：实现盲注（小盲/大盲）自动扣除、底池金额实时累加、加注时的筹码动画飞入底池。
2. 推进后续发牌流程：当一轮下注平衡后，依次发出翻牌（3张）、转牌（1张）、河牌（1张），并更新公共牌区域。
3. **开发 AI 性格决策引擎（重点）：**
    - **性格模板定义：** 为 AI 预设差异化性格标签及底层决策权重（VPIP 入池率、PFR 翻前加注率等）。
        - **紧弱型（岩石玩家）：** 极度保守，仅顶级强牌（AA/KK/AK）入池，面对加注弃牌率 >90%。
        - **松凶型（疯狂玩家）：** 极具侵略性，入池率 >23%，频繁加注与半诈唬，善于利用位置施压。
        - **狡猾型（欺诈大师）：** 高诈唬频率，擅长根据公共牌面（如听牌面）进行反逻辑下注，或强牌慢打。
        - **跟注站（被动玩家）：** 极度厌恶弃牌，有一线成牌希望即跟注到底，极少主动诈唬。
    - **记忆与反馈机制：**
        - **短期情绪模拟：** 连续输掉底池会触发"上头（Tilt）"状态，导致后续几局非理性激进。
        - **长期对手画像：** 记录玩家行为（如频繁诈唬），狡猾型 AI 会相应提高抓诈唬概率。
4. **AI 拟人化交互表现：**
    - **差异化思考时间：** 紧弱型拿强牌秒跟，遇复杂牌面犹豫；狡猾型强牌故意拖延，诈唬反而快速下注。
    - **表情与聊天互动：** 结合性格触发待机动画与聊天文本（如松凶型诈唬成功后播放嘲讽表情并发送"你抓不到我！"）。

### Phase 4: 游戏完善、音效与本地化体验

**目标：** 提升游戏的沉浸感与耐玩度，增加辅助功能与结算面板。

**具体任务：**

1. 增加音效系统：引入洗牌声、发牌声、筹码碰撞声、赢家庆祝音效，并添加全局静音开关。
2. 开发结算弹窗：每局结束后弹出详细战报，展示获胜者的牌型名称（如"两对"、"同花"），并列出所有参与比牌玩家的最终手牌。
3. 增加游戏辅助功能：实现"重新开始游戏"（重置所有玩家筹码）、"离开牌桌"功能。
4. 破产与救济机制：当玩家筹码归零时，弹出提示并提供"领取救济金"或"重新开始"的选项，确保单机体验不会因破产而强行中断。
5. 移动端适配优化：调整牌桌、手牌与操作按钮在移动设备屏幕上的布局与触控热区，确保竖屏或横屏下的良好体验。

## 非功能需求

### 性能要求

- 游戏启动加载时间不超过 3 秒
- 发牌动画帧率保持在 60fps 以上
- AI 决策计算时间不超过 1 秒

### 兼容性要求

- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 支持移动端（iOS、Android）竖屏和横屏模式

### 安全性要求

- 所有游戏逻辑在客户端完成，无需网络验证
- 本地存储玩家游戏进度和筹码数量
